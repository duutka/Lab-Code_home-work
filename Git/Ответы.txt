1.Как инициализировать репозиторий? Как можно склонировать репозиторий на локальный компьютер?
Для инициализии используется команда git init. 
Чтобы склонировать репозиторий git clone "ссылка на репозиторий" *-b "имя бранча"*(*-необязательно)

2. В чем разница между локальным и удаленным репозиторием? Как можно привязать локальный репозиторий к
удаленному?
Локальный репозиторий это то, что находится на локальной машине. Мы можем весь код "запушить" в удаленный репозиторий и
другие могут скачать наши доработки. Для связки мы должны склонировать репозиторий.

3. Что делаю команды add, commit и push?
add - добавляет "коммит" в статус Stage(подготовка)
commit - совершаем "коммит" всего из статуса Stage(также добавляем описание). "Пушим" в локальный репозиторий
push - "пушим" коммит в удаленный репозиторий

4. Как обновить локальный репозиторий из удаленного? Как обновить сразу локальный репозиторий и ветку на которой
вы находитесь с удаленного репозитория?
Мы можем написать git fetch(закидываем в локальный репозиторий) и git pull(чтобы изменения сразу были в текущей ветке), 

5. Что делают команды reset и revert? Когда можно использовать reset, а когда нужно использовать revert?
Команды reset и revert отменяют коммиты. 
Когда коммит уже в удаленном репозитории делать revert. Это обратный коммит.
Иначе использовать reset.

6. Что делают команды merge и rebase? В чем разница между ними? Когда рекомендуется использовать merge, а
когда rebase?
merge - создает коммит, у которого два родителя(между котороми происходит слияния). Используется, когда в ветки работают 
несколько человек. Чтобы не потерять коммиты.
rebase - копирует набор коммитов и выносит их в другое место. Используется, когда вы один работаете в ветке.

7. Как взять любой коммит из любого места репозитория в текущую ветку?
git cherry-pick "коммит1" "коммит2" ...
Команда копирует коммиты на место, где сейчас находишься
